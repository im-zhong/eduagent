1. 国内外研究现状及选题意义
在当前全球科技竞争加剧和数字化转型加速推进的新形势下，人工智能已成为引领经济社会深刻变革的核心驱动力。党的二十大以来，以习近平同志为核心的党中央把发展人工智能提升到国家战略高度，明确提出要在教育等关键领域加快推进人工智能的深度应用。
在人工智能与教育深度融合的进程中，智能出题作为典型应用场景之一，正成为破解教育供给侧效率瓶颈的重要突破口。近年来，大型语言模型（LLM）作为最先进人工智能的代表，由于其强大的自然语言理解能力和LLM智能体任务自动化特性，正在推动形成智能教育新范式，有力提升教育资源的生成效率。这为长期困扰教育领域的出题效率低下问题提供了新的解决方案，能够有效缓解传统人工命题模式中人力物力消耗大、教师工作负担重等问题。
国内高校和教育科技企业正在探索LLM出题智能体建设，如江西师范大学提出一种基于大语言模型的自动出题方法和系统，其构建“知识点-能力-题型”三维矩阵，为学生提供合适的习题，提高了题目考察的灵活程度，提高了学习效率和测评效率；中国科学技术大学提出了选择性命题推理系统及方法，其以提示词工程方法为基础，设计了新的推理框架与模块，从而能够提高大语言模型解决自然语言表述的复杂推理问题的能力；中国人民解放军军事科学院系统工程研究院提出了一种基于大模型的形式化定理证明生成方法及系统，其可以大大降低自动生成定理证明的复杂度；华南理工大学出版社2025年推出的自主可控AI数字出版平台，通过私有大模型底座实现教材机器人动态答疑与练习生成；猿辅导、作业帮等平台在 K12 场景下尝试通过模型辅助布题；阿里云助力金数据打造新时代的AI考试系统等，标志着题目生成进入智能化新阶段。
国际教育出题领域烨正加速AI技术融合，德国FelloFish平台实现知识点自动标注但难以处理主观题生成；美国Knewton系统依赖历史题库数据，对新教材适配能力弱；英国Merlyn Mind 教育题库生成系统，其设计并评估Transformer based模型来自动生产校内测 MCQ，并经教师调研验证其有效性。与此同时，以Toolsaday、OpExams、Quillionz为代表的平台已将AI出题转化为规模化服务，标志着该领域完成从技术探索到产业成熟的范式跃迁。
    通过对以上现状分析发现，AI大语言模型出题方面存在以下实际问题或挑战：
（1）知识深度不足：现有教材知识抽取工具多局限于表层概念提取，未建立知识点-能力目标-常错点的三维关联，导致生成题目与教学核心素养脱节；
（2）生成可控性弱： 如上海教育出版社AI出题系统存在语言风格偏离学段、干扰项机械重复等问题，缺乏对解题认知负荷的量化控制；
（3）动态适配缺失：传统系统无法随新课标实时更新策略，造成内容滞后性，难以满足分层教学需求；
（4）样本依赖性强：现有方法过度依赖大规模历史题库微调模型，导致在新教材适配任务中泛化能力不足，尤其面临"零样本"教材对齐需求时迁移困难、内容质量失控。
（5）教情学情协同闭环缺失：现有系统界面未能建立教学决策与学习反馈的闭环通路，教师无法依据实时学情动态调整教学策略，学生难以获取个性化习题反哺。
    针对以上挑战，课题提出一种基于GLM系列大模型构建“AI出题智能体”解决方案，其通过GLM文档解析API，采用RAG多阶段优化策略构建教材知识三元组图谱，避免现有教材知识抽取工具抽取知识深度不足问题；通过搭建嵌入教育约束规则的智能体工作流能解决现有AI智能出题智能体对于题目生成可控性弱、动态适配缺失与样本依赖性强的问题；通过教师学生协同平台能避免教情学情协同闭环缺失的挑战。因此，本解决方案可在无大规模样例题微调情况下，生成高质量、学段契合、知识点匹配、干扰项合理的客观题（如选择题）和主观题，突破传统题库依赖模式，建立教情学情协同闭环，实现教材内容到习题的智能转化。

2.1 研究目标
本课题总体目标是建立一种新的基于GLM大模型的AI出题智能体解决方案，具体研究内容和技术路线如图1所示：
 
图1 基于GLM大模型的AI智能出题智能体的研究内容与技术路线
    依据图1，着眼课题需求，为解决在无大规模样例题微调情况下，生成高质量、学段契合、知识点匹配、干扰项合理的客观题（如选择题）和主观题的实际问题，本课题研究内容包括教材多模态知识解析、教育约束下的题目生成和教师学生协同平台，具体描述如下：
研究内容一：教材多模态知识解析    
    从图文混排教材中提取知识点、能力目标及典型错误模式。针对教材图文混排特性，利用GLM-4.1V-Thinking大模型的跨模态理解能力，实现教材内容的深度语义挖掘。重点突破知识点、能力目标、常错点三要素的协同提取：如通过视觉-文本对齐技术解析数学公式与物理图示的隐含逻辑；构建知识关联规则引擎，建立"知识点-能力目标-典型错误"三维知识图谱。该研究目标为题目生成提供底层数据支撑，实现无微调场景下的教材知识数字化。
研究内容二：教育约束下的题目生成
    支持客观题/主观题的风格适配难度控制；创新干扰项生成机制，模拟学生实际的错误路径，设计合理的干扰项；设计“解题负担”评估模型，根据学段特性生成解题步骤。该研究确保生成题目严格匹配课标要求，在零样例微调条件下产出学段契合、干扰项合理的高质量客观题与主观题。
研究内容三：教师学生协同平台
    提供教师端和学生端，构建双端联动的教育平师端提供题目审核工作台与学情查询中心等功能，支持题目修正、个人或班级学情查询；学生端部署习题练习中心与错题解析中心，基于学生历史错题动态生成习题以及错题解析，通过错因溯源和靶向训练形成"练习-诊断-巩固"闭环。该平台实现教师经验与AI能力的协同进化，推动教育出版从静态资源向动态服务转型。
本课题技术路线基于多模态大模型和知识图谱驱动的智能教育技术架构，包括GLM-4.1V-Thinking图文混合理解、基于多阶段RAG的知识提取、三维知识图谱建模、题目难度控制、认知合理的干扰项生成、基于Streamlit的前端页面制作、基于FastAPI的后端API服务搭建、基于LangChain的Agent工作流搭建。具体内容描述如下。
（1）基于GLM大模型的图文混合理解   GLM-4.1V-Thinking是智谱2025年7月开源的多模态大模型，通过视觉-语言联合推理架构实现教材图文深度解析，具体架构如图2所示。其核心创新包括：动态分辨率适应：采用双三次插值动态调整位置编码，支持任意分辨率的教材扫描件（如4K图表），避免传统OCR的语义割裂问题；跨模态对齐：GLM-4.1V支持文本图片双模型，可精准关联数学公式与解题文本等需要图文混合理解的场景；思维链推理（CoT）：模型支持“深度思考”模式，可以模拟教师备课思维，从教材中分层提取“知识点-能力目标-常错点”三元组。该技术无需微调即可处理图文混排教材，为知识图谱构建提供底层支撑。
（2）基于多阶段RAG的知识提取   基于多阶段RAG的知识提取技术旨在解决从复杂教材文档中精准、深度获取结构化知识（如知识点、能力目标、常错点）的技术难点。不同于传统RAG一次性检索与生成，该技术采用分层递进的策略。首先进行粗粒度的文档级检索，快速锁定与目标知识相关的教材章节或页面范围。然后在此范围内进行细粒度的段落级检索，聚焦包含核心知识点描述、能力要求或典型错误分析的具体文本片段。最后，结合预定义的知识抽取prompt，对这些精选出的上下文片段进行深度语义解析，精确抽取出知识点实体、对应的认知能力目标要求以及常见的错误模式及其成因。这种多阶段方法显著提高了知识提取的准确性和针对性，有效降低了无关信息的干扰，为构建高质量的三维知识图谱提供了可靠的数据源。
 
图2 GLM-4.1V-Thinking架构图
（3）基于教育场景的三维知识图谱构建   三维知识图谱建模技术致力于构建一个能够深度刻画教材知识内在教学逻辑的结构化知识库。它区别于传统仅关注知识点本身的知识图谱，创新性地将三个核心教学维度——知识点本身、该知识点要求学生达到的能力目标（如记忆、理解、应用、分析等）、以及学生学习该知识点时常犯的典型错误及其背后的错误思路——进行有机整合和关联建模。具体来说，该技术首先将抽取出的知识点作为核心节点，然后为每个知识点明确关联其对应的能力目标要求节点，并进一步链接到与之相关的、经过归纳总结的典型错误模式节点。最终形成一个立体的网络结构，清晰展现知识点掌握需要哪些能力支撑，以及哪些错误认知路径可能阻碍这种掌握。这种三维建模为后续题目生成提供了精准的语义基础，确保生成的题目能有效覆盖目标知识点、针对性地考察特定能力层级，并设计出模拟真实错误思路的合理干扰项。
（4）基于参数调控算法的题目难度控制   题目难度控制技术致力于精确调节AI生成题目的挑战程度，使其严格匹配目标学段学生的认知水平。该技术通过综合分析多个影响难度的关键因子来实现这一目标：首先是识别题目所考查的知识点在三维知识图谱中对应的认知能力层级（如记忆、理解、应用或分析），层级越高通常难度越大；其次是预估并控制解题所需的逻辑步骤数量与复杂度，步骤越多、推理链条越长则难度提升；最后是融入对“解题负担”的建模，即根据学段标准限制题目涉及的知识点范围、计算量或思维跳跃性，避免超出学生合理的能力负荷。该技术允许用户教师获学生通过预设的难度等级（如简单、中等、困难）或更精细的参数滑块（如步骤数上限、知识点深度）来动态约束题目的生成过程，确保在零样例微调条件下也能稳定产出难度适配、负担合理的高质量题目。
（5）基于认知合理算法的干扰项生成 认知合理的干扰项生成技术旨在为选择题创造出既具有迷惑性又符合学生真实认知错误路径的错误选项。它摒弃了随机生成或简单变形等的方法，而是紧密依托于三维知识图谱中归纳的“典型错误模式”。该技术首先分析目标知识点所关联的常见学生误解、易混淆概念以及典型的计算或推理错误。然后，在生成干扰项时，系统性地模拟这些真实发生的错误思路：例如，针对一个物理公式题，干扰项可能源自学生对公式适用条件的误解或单位换算的疏忽；针对一个历史事件题，干扰项可能源于对时间线索的混淆或因果关系的颠倒。通过这种对学生学习过程中产生的错误认知的有效建模，生成的干扰项能有效诊断学生的薄弱环节，而不仅仅是增加猜测难度，从而显著提升题目在诊断性评估和针对性练习中的价值。
（6）基于Streamlit的前端页面制作   Streamlit是一个开源的Python库，它允许开发者用简单的Python脚本快速构建和部署交互式的数据应用网页，特别适合机器学习和数据科学。在本解决方案中，Streamlit主要用于构建教师端和学生端平台的用户界面。利用Streamlit的特性，通过编写Python代码即可高效地创建出包含文本展示区（如题目呈现、解析内容）、交互控件（如下拉菜单选择学段和难度、滑块调节参数、按钮触发生成或提交）、数据可视化（如学情分析图表）以及文件上传下载（如教材导入、题目导出）等功能模块的网页应用。这种技术极大地简化了前端开发流程，使得非专业前端工程师也能快速搭建出功能完备、用户友好的教育平台界面，服务于教师的题目审核、学情查询以及学生的习题练习、错题分析等核心功能。
（7）基于FastAPI的后端API服务  FastAPI 是一个现代、高性能的 Python Web 框架，特别适合构建 API。它以开发速度快、运行效率高、自动生成交互式 API 文档以及内置数据验证而著称。在本解决方案中，FastAPI 用于构建支撑整个智能出题平台的核心服务层。利用 FastAPI 创建一系列 API 接口，负责处理来自 Streamlit 前端的关键请求。例如，接收教师上传的教材文件并触发知识提取流程；响应前端请求调用 GLM 大模型生成符合难度和风格要求的题目；提供学生错题数据以生成个性化练习题和解析报告等。 FastAPI 确保这些请求能够被高效、可靠地路由到相应的处理模块（如知识图谱引擎、题目生成智能体、学情分析模块），并将处理结果（如生成的题目、知识图谱片段、学情分析数据）安全地返回给前端应用。FastAPI为整个AI出题智能体解决方案提供了稳定、可扩展和高性能的后端支撑。
（8）基于LangChain的Agent工作流   LangChain 是一个用于构建大语言模型应用的开源框架。它提供了标准化的接口和模块化组件，简化了将大语言模型（如 GLM）与外部数据、工具以及复杂任务流程集成的过程。在本解决方案中，LangChain 主要用于构建出题智能体的工作流的核心逻辑链条。该技术将整个出题任务分解为一系列可管理的步骤（如知识图谱查询、难度参数应用、题目主干生成、干扰项设计、解题步骤规划等），并为每个步骤定义相应的工具（如访问三维知识图谱的检索器、难度控制计算器、访问智谱大模型API等 ）。利用 LangChain 的 Agent 工作流定义能力，出题智能体能够根据当前任务状态（如用户指定的知识点和难度）和预定义prompt，动态地决定调用哪些工具、以什么顺序执行这些步骤，并处理步骤间的信息传递。例如，出题智能体可能先查询知识图谱获取知识点和能力目标，接着应用难度参数控制解题步骤复杂度，再调用智谱GLM大模型生成题目主干，最后基于典型错误模式生成干扰项。这种工作流搭建确保了出题过程灵活、可控且自动化，将多个独立模块有机整合成一个智能的出题智能体。
2.2 研究方法
本课题采用需求分析、解决方案设计、智能体算法设计以及可视化交互来逐步实现课题目标。解决方案和智能体算法都通过可视化系统“教师学生协同平台”呈现出来。该系统的总体设计如图3所示。
为助力教育数字化转型，我们推出了一套融合多模态理解的AI智能出题智能体系统。该系统依托GLM-4.1V-Thinking大模型的核心能力，实现对教材内容的深度解析——从文图混合数据提取、例题习题结构化处理，到三维知识图谱构建（涵盖知识点、能力目标与常见错因）。在此基础上，系统支持约束条件下的题目自动生成、学情评估与个性化练习，覆盖备课、命题、诊断、巩固全流程。具体的系统功能模块描述如下：
（1）文图混合数据提取
功能目标： 利用GLM-4.1V-Thinking大模型直接从教材原始文件（扫描件/电子文档）中整体性提取并理解页面上的文本内容、插图及其内在关联，为后续知识抽取提供高质量基础数据。
功能描述： 调用GLM-4.1V-Thinking模型处理上传的教材文件。模型同时理解页面上的文字（含复杂公式）和插图（如数学图、物理示意图），分析它们之间的语义关系（如文字描述与对应插图、公式图表与例题步骤）。
输出结果： 按章节组织的结构化的知识点。
 
图3 基于GLM大模型的AI智能出题智能体功能层次方框图
（2）例题解析数据提取
功能目标： 从教材中识别和提取例题及其详细的解题步骤、方法思路和最终答案。
功能描述： 分析教材内容，定位标有“例”或类似标识的题目部分。提取题目题干、详细的解题过程（包括关键步骤、运用的公式或方法）、以及最终答案或结论。
输出结果： 结构化的例题数据集，每条记录包含题干、解题步骤列表、答案、所属章节/知识点标签。
（3）习题数据提取
功能目标： 收集教材章节末尾或习题集中提供的练习题题目及其参考答案。
功能描述： 识别并提取教材中明确标记为练习题或习题的题目内容，通常位于章节末尾或专门习题集。提取题目题干、选项和标准答案。
输出结果： 结构化的习题数据集，每条记录包含题干、选项、标准答案、所属章节/知识点标签、题型。
（4）构建知识点图谱
功能目标： 构建一个结构化的知识网络，清晰表示教材中知识点之间的层级和关联。
功能描述： 基于知识抽取的结果，将识别出的核心知识点作为节点，利用预定义规则或GLM模型分析知识点间的逻辑关系，建立节点间的连接。
输出结果： 一个图结构数据库或知识图谱文件，包含知识点实体及其相互关系，描述知识点的层次和关联。
（5）提取能力目标
功能目标： 明确标识教材中各个知识点要求学生掌握的具体认知能力层级，包括：记忆、理解、应用、分析等。
功能描述： 分析教材内容，将知识点映射到认知能力层级上。例如，某个概念可能要求“记忆”，而另一个共识则要求“应用”或“分析”，可以用来解决某个例题。
输出结果： 知识点与对应能力目标层级的关联列表或知识图谱中的属性标注（如知识点A：能力目标=应用）。
（6）提取常见错因
功能目标： 归纳总结学生在学习特定知识点时普遍存在的错误认知、误解或典型错误模式。
功能描述： 基于教材中提示的“易错点”、“注意”部分、例题解析中的错误示范，以及可能通过分析历史习题数据中正确率低的题目，归纳出错误的模式。提取并分类这些常见错误的原因描述。
输出结果： 知识点关联的常见错误模式列表或知识图谱中的属性标注（如知识点B：常见错因=混淆概念X与概念Y）。
（7）多模态教材解析
功能目标： 利用GLM-4.1V-Thinking模型深度解析教材内容，实现核心知识点、能力目标和典型错误模式三要素的协同抽取。
功能描述： 基于“文本和插图”提取的原始数据，调用GLM模型结合多阶段RAG策略，模拟教师备课思维，从教材中精准识别并抽取知识点实体、对应的能力要求层级以及相关的学生常见错误原因描述。
输出结果： 结构化的知识三元组数据（知识点、能力目标、常见错因），作为构建三维知识图谱的直接输入。
（8）教育约束题目生成
功能目标： 在教师设定的教学约束条件下（范围、题型、难度），利用大模型生成高质量、知识点匹配的题目（客观题/主观题）。
功能描述： 结合三维知识图谱和教师输入的要求（如章节范围、题型偏好、难度等级），利用大模型（如GLM）生成符合约束的题目主干。对于客观题，结合干扰项生成技术；对于主观题，规划合理的解题步骤。
输出结果： 生成的完整题目。包括：题干、选项/解题步骤框架、知识点标签、预估难度。
（9）题目难度控制
功能目标： 量化并精确控制生成题目的难度，使其匹配目标学段学生的认知水平。
功能描述： 分析题目涉及知识点的能力目标层级（越高越难）、解题所需逻辑步骤的复杂度与数量、以及知识点的深度与广度（控制解题负担）。允许教师通过参数（如难度等级、步骤数上限）进行调控。
输出结果： 对生成题目或已有题目进行难度评分（如1-5级），并在生成过程中作为约束条件影响模型输出。
（10）多阶段检索增强
功能目标： 利用检索技术从教材知识库中精准获取与当前任务，如知识点讲解、题目生成等最相关的信息片段。
功能描述： 在知识抽取和题目生成过程中，根据当前处理焦点（如特定知识点、教师指定的章节范围），从三维知识图谱、原始教材文本或习题库中检索最相关的上下文信息，如概念定义、典型例题、相关错因等，提供给大模型作为生成依据。
输出结果： 与当前任务高度相关的知识片段或数据记录，作为上下文输入大模型。
（11）认知合理干扰项生成
功能目标： 为选择题生成具有教学诊断价值的迷惑性干扰项，模拟学生真实错误思路。
功能描述： 基于三维知识图谱中目标知识点关联的“常见错因”，分析对应的典型错误认知模式。利用这些模式指导模型生成干扰项，使其反映学生可能犯的具体错误，如概念混淆、计算疏忽、推理偏差等，而非随意编造。
输出结果： 为选择题生成的多个错误干扰项，每个选项对应一种典型的错误模式。
（12）题目审核
功能目标： 为教师提供界面，对AI生成的题目内容、选项、难度等进行审阅、修改和批准入库。
功能描述： 教师在操作平台上查看AI生成的题目详情，包括题干、选项/解析、知识点、难度等。教师可对题目描述进行文字修改、调整选项、修正难度评级、添加批注说明，最终决定是否采纳该题目进入题库。
输出结果： 教师修改审核通过，并确认入库的题目；教师驳回的题目及反馈意见。
（13）学情评估
功能目标： 为教师提供查询和分析学生/班级学习情况的数据工具。
功能描述：教师可查询指定学生或班级的总体做题数据（数量、正确率）、按知识点或章节维度的掌握情况分析、常见错误类型统计。平台展示可视化报告，如正确率趋势图、知识点掌握热力图、错题分布等。
输出结果：学生/班级的学情统计报告和分析图表，帮助教师了解教学效果和薄弱环节。
（14）习题练习
功能目标：为学生提供个性化的习题练习功能，针对其薄弱点或指定范围生成题目。
功能描述：系统根据学生历史练习记录分析其薄弱知识点，或接受学生主动选择的章节/知识点范围，从题库中按难度分布抽取或实时生成匹配的题目集，可以按照试卷形式或单题模式，供学生作答。
输出结果：呈现给学生进行作答的习题集或单个题目。
（15）错题分析
功能目标：对学生练习中的错题提供智能解析和巩固练习，促进理解和掌握。
功能描述：学生完成练习后，系统自动标记错题。针对每道错题，调用大模型生成详细的智能解析，指出错误原因，并基于该错题知识点和错误模式，生成若干道相似的变式题供学生巩固练习。
输出结果： 错题的详细解析报告和配套的变式巩固练习题集。
